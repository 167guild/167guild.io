# Support

## Getting Help

- **Questions / usage help:** open a GitHub Discussion
- **Bug reports:** open a GitHub Issue using the appropriate template
- **Security concerns:** follow `SECURITY.md` and report privately

## Before Opening a Request

1. Check `README.md` and `docs/setup/README.md`.
2. Verify your `.env` values were copied from `.env.example`.
3. Run local checks:

```bash
task init
task lint
task up
```

## Operational Incidents

For production-impacting incidents, contact repository maintainers through private channels and include:

- Time of incident
- Observed symptoms
- Recent deployment/change context
- Logs or health-check output (without secrets)
