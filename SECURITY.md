# Security Policy

## Supported Versions

This repository is currently pre-1.0 and actively maintained on `main`. Security fixes are applied to the latest codebase.

## Reporting a Vulnerability

Please report vulnerabilities privately.

- Preferred: GitHub Security Advisories / private security report
- Alternative: direct maintainer contact through private repository channels

Do **not** open public issues for unpatched vulnerabilities.

## What to Include

- Affected component(s)
- Reproduction steps or proof-of-concept
- Impact assessment
- Suggested remediation (if available)

## Response Expectations

- Initial acknowledgement target: within 3 business days
- Triage and severity assessment target: within 7 business days
- Remediation timeline depends on severity and release risk

## In Scope (Examples)

- Secret leakage in committed files or workflows
- Authentication/authorization bypass
- Privilege escalation in Wiki.js role configuration
- Data exposure through misconfigured networking or public ports
- Command/script injection in deployment automation

## Out of Scope (Examples)

- Placeholder credentials in committed templates
- Vulnerabilities requiring physical/local-only access to contributor machines
- Third-party issues with no available patch or mitigation in this repository
